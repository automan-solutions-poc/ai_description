01 — The stack

One engine, three levels of independence

Agentic AI is built on AI agents, which are built on generative AI. Each layer adds a capability the one below it doesn't have.

GEN

Generative AI

"Can you create this for me?"

A model that turns a prompt into content — text, code, an image, audio, video. Powerful, but reactive: it does nothing until asked, and forgets everything once it answers.

BehaviorSingle prompt → single response. No tools, no memory, no goal-tracking.
AGT

AI Agent

"Can you complete this task using tools?"

A generative model wrapped in a loop. It can call APIs, run code, search the web, and decide what to do next based on what just happened — until the task is actually done.

BehaviorGoal-directed. Plans, acts, observes results, repeats — bounded to one task.
AGC

Agentic AI

"Can you run this whole workflow on your own?"

Multiple agents (or one very capable one) coordinating across an open-ended workflow — delegating sub-tasks, recovering from failure, and escalating to a human only when it truly must.

BehaviorProactive & autonomous. Sets sub-goals, orchestrates other agents, runs continuously.

03 — Under the hood

What's actually inside an agent

Four components, wired into a closed feedback loop. Get these right and every framework, paper, or diagram you encounter will make sense.

01 / CORE

LLM / Reasoning Engine

The "brain" that interprets a goal, weighs options, and decides the next step. GPT, Claude, Gemini, Llama.

02 / STATE

Memory

Short-term working context for the current task, plus long-term storage — past actions, facts, learned preferences.

03 / REACH

Tools

What turns "understanding" into "doing" — web search, code execution, APIs, file systems, CRMs.

04 / CONTROL

Planning & Orchestration

Breaks a goal into ordered sub-tasks, sequences them, and decides when the job is actually finished.

Reasoning patterns

How agents structure their thinking

ReAct — Reason + Act

The agent writes out its reasoning ("Thought") before each action, then reacts to what comes back. Slower and more token-hungry, but tightly grounded in real outcomes rather than a guess made up front.

Thought→ Action→ Observation→ Thought→ … loop

Plan-and-Execute

A capable "planner" model lays out the full sequence of sub-tasks up front. A cheaper, faster "executor" then works through each one and reports back. Strategy and tactics are split into two roles.

Planner: builds task graph→ Executor: step 1→ step 2→ step n

Reflexion / Self-Critique

Before moving on, the agent checks its own intermediate output for mistakes and revises its approach — catching errors mid-task instead of only discovering them at the end.

Act→ Self-critique→ Revise→ Act again

Hierarchical Decomposition

A single "monolithic" agent loses track of its own goal as context grows — a failure mode often called context dilution. The fix: a top agent plans and delegates, mid-level agents break work down further, and low-level agents execute narrow, tool-specific jobs.

High-level agent (plans)→ Mid-level agents (decompose)→ Low-level agents (execute)
PROTOCOL

MCP — Model Context Protocol

An open standard for connecting agents to outside tools and data — one consistent interface instead of a custom integration per API. Often called "the USB-C of agent tooling."

PROTOCOL

A2A — Agent-to-Agent Protocol

Standardizes how independent agents — even from different frameworks — discover and talk to each other, so a LangGraph agent and a CrewAI agent can collaborate in one network.

04 — Build with

Picking a framework

Each one reflects a different philosophy about how agents should be coordinated. None is universally "best" — fit the tool to the workflow shape.

FrameworkCoordination modelBest for
LangGraph Directed graph, shared state Production workflows needing audit trails, checkpointing, human-in-the-loop most control
CrewAI Role-based "crews" Fast prototyping, work that splits into specialist roles lowest learning curve
Microsoft Agent Framework Multi-agent conversation Azure/Microsoft-native enterprise, agents debating or reviewing each other
OpenAI Agents SDK Handoff-based Fast GPT-centric builds with sandboxing and built-in guardrails
Google ADK Hierarchical agent tree Multimodal agents, Google Cloud / Gemini-native deployments
Claude Agent SDK Tool-use + native MCP Safety-first agents, computer-use tasks native MCP
05 — In the wild

What each layer is actually used for

Click a row to expand it.

GEN Generative AI

  • Content marketing — blog drafts, ad copy, product descriptions at scale
  • Software development — code completion, refactoring, test generation
  • Design & creative — concept art, mockups, music and video drafts
  • Education — personalized explanations, practice questions, study guides

AGT AI Agents

  • Customer support agents that read a ticket, check order history via API, and resolve it end-to-end
  • Coding agents that read a repo, write a patch, run tests, and open a pull request
  • Research agents that search the web, read multiple sources, and compile a structured brief
  • Data agents that query a database, run analysis, and generate a report or chart

AGC Agentic AI / Multi-Agent

  • Procurement automation — agents that monitor inventory, negotiate with supplier systems, place orders within approved limits
  • HR onboarding pipelines — a chain of agents provisioning accounts, scheduling training, answering new-hire questions
  • Compliance & audit workflows — agents that continuously monitor transactions, flag anomalies, draft reports for review
  • Autonomous research ("autoresearch") — agents that design experiments, collect data, and summarize findings with minimal steering
  • DevOps/CI-CD pipelines — agents that debug failing builds, write fixes, run tests, and open pull requests automatically
06 — Read before you ship

Risk scales with autonomy

Generative AI introduces informational risk — bad text a human reviews. Agentic AI introduces operational risk — wrong actions on live systems, sometimes before anyone sees them.

Shared risks (generative core)

  • Hallucination — confident but factually incorrect output
  • Bias — models can reproduce patterns in training data
  • Copyright & IP — generated content can closely resemble existing work
  • Overreliance — trusting output without verification

Amplified by autonomy

  • Operational risk — a real, possibly irreversible action taken before review
  • Context dilution — long tasks cause the agent to lose track of the original goal
  • Runaway loops — poor bounds drive up cost/latency without converging
  • Expanded attack surface — live-system + open-web access invites prompt injection
  • Governance gaps — unclear ownership when an autonomous system errs

Rule of thumb: start with the lowest autonomy level that actually solves the problem. Add a human-in-the-loop checkpoint before any high-impact or irreversible action, and log every decision an agent makes for auditability.

08 — Reference

Glossary

Search any term used throughout this guide.

LLM (Large Language Model)
A neural network trained on vast text data to understand and generate human-like language.
Prompt
The input instruction or question given to a generative AI model.
Token
A chunk of text — often a word or part of one — that a model processes as a unit.
Hallucination
Confident but factually incorrect output produced by a generative model.
RAG (Retrieval-Augmented Generation)
A technique where a model retrieves relevant external information before generating a response.
Function Calling / Tool Use
The mechanism by which an LLM decides to invoke an external tool or API and use its result.
ReAct
An agent architecture pattern that interleaves explicit reasoning ("Thought") with actions and observations.
Context Window
The amount of text/tokens a model can consider at once when generating a response.
MCP (Model Context Protocol)
An open standard for connecting AI agents to external tools and data sources.
A2A (Agent-to-Agent Protocol)
An open standard letting independent agents, even from different frameworks, discover and communicate with each other.
Human-in-the-Loop (HITL)
A design pattern where a human must review or approve certain agent actions before they proceed.
Multi-Agent System
An architecture where multiple specialized agents collaborate, each handling part of a larger goal.
Autonomy Ladder
A framework describing increasing levels of AI independence, from simple prompt-response up to fully autonomous multi-agent systems.
Context Dilution
A failure mode where an agent loses track of its original goal as a long task accumulates intermediate steps.
